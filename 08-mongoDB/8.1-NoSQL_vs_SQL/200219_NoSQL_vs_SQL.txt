* NoSQL vs NoSQL
    + 몽고디비(NoSQL)
        - 고정된 테이블이 없음
            1) MySQL(SQL)은 users 테이블을 만들 때 name, age 등 컬럼과 자료형 옵션을 정의
            2) 몽고디비(NoSQL)는 users 컬렉션을 만들고 끝. 
                users 컬렉션에는 어떠한 데이터든 다 들어갈 수 있음.
        - 몽고디비에는 MySQL과 달리 JOIN 기능이 없음
        - 트랜잭션을 지원하지 않음
            [?] 트랜잭션: 여러 쿼리가 모두 정상적으로 수행되거나 아예 하나도 수행되지 않음을 보장하는 기능
            [!] 몽고디비 4 버전부터는 트랜잭션을 지원. (2020.02.19 기준, 최신버전 4.2.3)
                - 몽고디비 트랜잭션에 대해 따로 알아보기. 책에는 없음.
        - 데이터의 일관성을 보장해주는 기능이 약한 대신 데이터를 빠르게 넣을 수 있음.
            쉽게 여러 서버에 데이터 분산할 수 있음.
        - 용어의 차이
            1) MySQL은        테이블, 로우, 컬럼
            2) 몽고디비에선     컬렉션, 다큐먼트, 필드

    + SQL과 NoSQL은 각각 특징이 다르므로 알맞은 곳에 사용하길.
        ex) 항공사 예약 시스템
            - 비행기 표에 관한 정보가 모든 항공사에 일관성있게 전달되어야 하므로 
                예약처리 부분의 데이터베이스는 MySQL 사용   
            - 핵심 기능 외의 빅테이터, 메세징, 세션 관리등에는 확장성과 가용성을 위해 몽고디비 사용
