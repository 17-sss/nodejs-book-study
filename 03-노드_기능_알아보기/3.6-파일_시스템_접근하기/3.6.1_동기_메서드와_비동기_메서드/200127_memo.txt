[!!] 동기와 비동기, 블로킹과 논블로킹

* 동기와 비동기: 함수가 바로 return되는지 여부
    - 동기: 바로 return X
    - 비동기: 바로 return O

* 블로킹과 논블로킹: 백그라운드 작업 완료 여부
    - 블로킹: 백그라운드 작업 상태를 계속 확인함.
    - 논블로킹: 백그라운드 작업 완료 여부를 확인하지 않음.

* 관계도 및 설명
    1. [동기 - 블로킹] 
        백그라운드 작업 완료 여부를 계속 확인하며, 호출한 함수가 바로 return되지 않고
        백그라운드 작업이 끝나야 return 됨.
    2. [비동기 - 논블로킹]
        호출한 함수가 바로 return되어 다음 작업으로 넘어가고, 백그라운드 작업 완료 여부는 
        신경 쓰지 않고 나중에 백그라운드가 알림을 줄 때 처리.
